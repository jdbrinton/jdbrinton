Overview

  • Geometry: Blunt cylinder (diameter = 1 m, length = 5 m)

  • Simulation type: Steady-state, turbulent flow
  • Solver: For compressible flow at Mach 0.5, we'll use rhoSimpleFoam

Detailed Steps

1. Set Up the Case Directory Structure

Create a directory for your case, e.g., Mach0p5Cylinder, and set up the standard OpenFOAM directory structure:

<BASH>
Mach0p5Cylinder/
├── 0
├── constant
│   ├── polyMesh
│   └── triSurface
└── system

2. Create the Cylinder Geometry

We'll use a geometry file in STL format for the cylinder.

a. Create the Cylinder STL File

Create a file named cylinder.stl inside constant/triSurface/ with the following content (use a CAD tool or script to generate an accurate STL file):

<details> <summary><strong>Click to view</strong> a simple OpenSCAD script to generate the STL</summary>

You can use OpenSCAD to create the cylinder:

<SCAD>
$fn=50; // Number of facets
cylinder(h=5, r=0.5); // Height = 5 m, Radius = 0.5 m
  • Save this script as cylinder.scad.
  • Export it as cylinder.stl by going to Design → Export as STL.
  • Place cylinder.stl in constant/triSurface/.
</details>

b. Generate the Base Mesh Using blockMesh

Create the file system/blockMeshDict:

<CPP>
/*--------------------------------*- C++ -*----------------------------------*\
| OpenFOAM Block Mesh Dictionary

    (-5 -5 -5)
    (10 -5 -5)
    (10 5 -5)
    (-5 5 -5)
    (-5 -5 5)
    (10 -5 5)
    (10 5 5)
    (-5 5 5)
);

blocks
(
    hex (0 1 2 3 4 5 6 7) (150 75 75) simpleGrading (1 1 1)
);

edges
(
);

boundary
(
    inlet
    {
        type patch;
        faces
        (
            (0 4 7 3)
        );
    }
    outlet
    {
        type patch;
        faces
        (
            (1 2 6 5)
        );
    }
    walls

            (3 2 6 7) // Top

);

mergePatchPairs

c. Configure snappyHexMesh to Refine Around the Cylinder

Create system/snappyHexMeshDict:

<CPP>
/*--------------------------------*- C++ -*----------------------------------*\
| OpenFOAM SnappyHexMesh Dictionary
\*---------------------------------------------------------------------------*/

castellatedMesh true;
snap            true;
addLayers       false; // Set to true if you want boundary layers

geometry
{
    cylinder.stl
    {
        type triSurfaceMesh;
        name cylinder;
    }
};

castellatedMeshControls
{
    maxLocalCells 1000000;
    maxGlobalCells 2000000;
    minRefinementCells 0;
    nCellsBetweenLevels 3;

    // Surface features
    features
    (
    );

    // Surface refinement
    refinementSurfaces
    {
        cylinder
        {
            level (2 3);
        }
    }

    resolveFeatureAngle 30;

    refinementRegions
    {
    }

    locationInMesh (0 0 0); // A point inside the fluid region
}

snapControls
{
    nSmoothPatch 3;
    tolerance 4.0;
    nSolveIter 30;
    nRelaxIter 5;
}

addLayersControls
{
    // Configuration for boundary layers (optional)
}

meshQualityControls
{
    maxNonOrtho 65;
    maxBoundarySkewness 20;
    maxInternalSkewness 4;
}

3. Generate the Mesh

In the terminal, navigate to your case directory and run:

<BASH>
blockMesh
surfaceFeatureExtract
snappyHexMesh -overwrite
  • blockMesh: Generates the base mesh.
  • surfaceFeatureExtract: Extracts edges from the STL file for refinement.
  • snappyHexMesh: Refines the mesh around the cylinder.

Check the mesh quality:

<BASH>
checkMesh

4. Define Boundary and Initial Conditions

a. Create Initial Condition Files in the 0 Directory

i. 0/U (Velocity Field)
<CPP>
/*--------------------------------*- C++ -*----------------------------------*\
| Velocity Field
\*---------------------------------------------------------------------------*/
FoamFile
{
    version     2.0;
    format      ascii;
    class       volVectorField;
    location    "0";
    object      U;
}

dimensions      [0 1 -1 0 0 0 0];

internalField   uniform (170 0 0); // Mach 0.5 at sea level (~170 m/s)

boundaryField
{
    inlet
    {
        type            fixedValue;
        value           uniform (170 0 0);
    }
    outlet
    {
        type            zeroGradient;
    }
    walls
    {
        type            noSlip;
    }
}
ii. 0/p (Pressure Field)
<CPP>

5. Configure Simulation Parameters

a. constant/thermophysicalProperties

<CPP>
/*--------------------------------*- C++ -*----------------------------------*\
| Thermophysical Properties
\*---------------------------------------------------------------------------*/

thermoType
{
    type            hePsiThermo;
    mixture         pureMixture;
    transport       sutherland;
    thermodynamics  hConst;
    equationOfState perfectGas;
    specie          specie;
    energy          sensibleEnthalpy;
}

mixture
{
    specie
    {
        nMoles      1;
        molWeight   28.97; // Air molar mass in g/mol
    }
    thermodynamics
    {
        Cp          1005; // Specific heat capacity at constant pressure
        Hf          0;
    }
    transport
    {
        As          1.458e-6;
        Ts          110.4;
    }
}

b. constant/turbulenceProperties

<CPP>
/*--------------------------------*- C++ -*----------------------------------*\
| Turbulence Properties
\*---------------------------------------------------------------------------*/

simulationType  RAS;

RAS
{
    RASModel        kOmegaSST;
    turbulence      on;
    printCoeffs     on;
}

c. system/controlDict

<CPP>
/*--------------------------------*- C++ -*----------------------------------*\
| Control Dictionary
\*---------------------------------------------------------------------------*/

application     rhoSimpleFoam;

startFrom       latestTime;

startTime       0;

stopAt          endTime;

endTime         1000;

deltaT          1;

writeControl    timeStep;

writeInterval   100;

purgeWrite      0;

writeFormat     ascii;

writePrecision  6;

writeCompression off;

timeFormat      general;

timePrecision   6;

runTimeModifiable true;

d. system/fvSchemes

<CPP>
/*--------------------------------*- C++ -*----------------------------------*\
| Finite Volume Schemes
\*---------------------------------------------------------------------------*/

ddtSchemes
{
    default         steadyState;
}

gradSchemes
{
    default         Gauss linear;
}

divSchemes
{
    div(rho*phi,U)      Gauss upwind;
    div(phi,k)          Gauss upwind;
    div(phi,omega)      Gauss upwind;
    div(phi,T)          Gauss upwind;
    div(phi,epsilon)    Gauss upwind;
    div(((rho*nuEff)*dev(T(grad(U))))) Gauss linear;
}

laplacianSchemes
{
    default         Gauss linear limited 0.333;
}

interpolationSchemes
{
    default         linear;
}

snGradSchemes
{
    default         limited 0.333;
}

fluxRequired
{
    default         no;
    p_rgh;
    pcorr;
}

e. system/fvSolution

<CPP>
/*--------------------------------*- C++ -*----------------------------------*\
| Finite Volume Solution
\*---------------------------------------------------------------------------*/

solvers
{
    p_rgh
    {
        solver          PCG;
        preconditioner  DIC;
        tolerance       1e-6;
        relTol          0;
    }
    U
    {
        solver          smoothSolver;
        smoother        GaussSeidel;
        tolerance       1e-5;
        relTol          0;
    }
    "(k|omega|epsilon|h|T)"
    {
        solver          smoothSolver;
        smoother        GaussSeidel;
        tolerance       1e-5;
        relTol          0;
    }
}

PISO
{
    nCorrectors     2;
    nNonOrthogonalCorrectors 0;
    pRefCell        0;
    pRefValue       0;
}

6. Run the Simulation

Ensure that all files are correctly set up. Then, in the terminal, execute:

<BASH>
rhoSimpleFoam
  • Monitor the residuals to ensure convergence.
  • The simulation may take some time depending on mesh size and computational resources.

7. Visualize the Results

a. Open the Results in ParaView

  • Launch ParaView by running:

    <BASH>
    paraFoam

  • If paraFoam doesn't work, you can use:

    <BASH>
    foamToVTK

    And then open the VTK files in ParaView.

b. Visualize Flow Parameters

  • Pressure Distribution: Display the pressure field to observe pressure variations around the cylinder.
  • Velocity Field: Visualize velocity vectors or magnitude to see the flow pattern.
  • Streamlines: Add streamlines to illustrate flow paths.
  • Slices: Create slices through the domain to inspect local flow properties.

Considerations

  • Mesh Resolution: Ensure that the mesh is fine enough around the cylinder to capture boundary layer effects and flow separation.
  • Turbulence Model: The kOmegaSST model is suitable for capturing adverse pressure gradients and separation.
  • Compressibility Effects: At Mach 0.5, compressibility effects are moderate but should be included.

Optional Enhancements

  • Boundary Layer Mesh Refinement: Enable layering in snappyHexMeshDict to better resolve the boundary layer.
  • Transient Simulation: Use rhoPimpleFoam if you want to perform a transient analysis.
  • Validation: Compare your results with analytical or experimental data to validate the simulation.
