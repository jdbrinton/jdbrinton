It sounds like gdb wasn't built during your installation of the RISC-V GNU toolchain. This is a common issue that usually happens when certain dependencies required by gdb are missing from your system. Let's work through the steps to resolve this problem.

1. Install Required Dependencies

Make sure your system has all the necessary packages installed for building the RISC-V GNU toolchain, including gdb. The key dependency you're likely missing is libexpat-dev, which is needed for gdb. Here's how you can install all the required dependencies:

For Ubuntu/Debian-based systems:

<BASH>
sudo apt-get update
sudo apt-get install -y \
  autoconf automake autotools-dev curl libmpc-dev libmpfr-dev libgmp-dev \
  gawk build-essential bison flex texinfo gperf libtool patchutils bc \
  zlib1g-dev libexpat1-dev

For Fedora/RHEL-based systems:

<BASH>
sudo dnf install -y \
  autoconf automake libmpc-devel mpfr-devel gmp-devel gawk bison flex texinfo \
  patchutils gcc gcc-c++ zlib-devel expat-devel

For Arch Linux:

<BASH>
sudo pacman -S --needed \
  autoconf automake libmpc mpfr gmp gawk bison flex texinfo \
  patchutils base-devel zlib expat

2. Clean Previous Builds

Since you've already run make, it's a good idea to clean the previous build to avoid any conflicts:

<BASH>
cd ~/riscv-gnu-toolchain/
make clean

3. Reconfigure and Build the Toolchain

Now, reconfigure the toolchain and start the build process anew:

<BASH>
./configure --enable-multilib --prefix=/opt/riscv --with-arch=rv32i --with-abi=ilp32
make

Note: The build process can take quite some time, so please be patient.

4. Verify the Installation of GDB

After the build process completes, check if gdb was successfully installed:

<BASH>
/opt/riscv/bin/riscv32-unknown-elf-gdb --version

You should see the version information for gdb if it was installed correctly.

5. If GDB Still Isn't Installed

If you still don't see gdb in /opt/riscv/bin/, try building the toolchain specifically for bare-metal (embedded) development, which includes newlib and gdb:

<BASH>
make newlib

6. Update Your PATH Environment Variable

Don't forget to update your PATH so that your shell can find the RISC-V tools:

<BASH>
export PATH="/opt/riscv/bin:$PATH"

You might want to add this line to your ~/.bashrc or ~/.bash_profile to make it permanent.

Additional Tips

  • Check for Build Errors: If the build process fails or you notice any errors, please check the build logs for any messages related to missing dependencies or configuration issues.

  • Ensure Correct Permissions: If you're installing to /opt/, you might need superuser permissions. Either run the build process as a regular user and install to a directory where you have write permissions or use sudo when necessary.

  • Alternate Configure Options: While not typically required, you can explicitly enable gdb in the configure script:

    <BASH>
    ./configure --enable-multilib --prefix=/opt/riscv --with-arch=rv32i --with-abi=ilp32 --enable-gdb

Troubleshooting Common Issues

  • Missing Expat Library: The most common cause for gdb not being built is the absence of the libexpat development library. Ensure libexpat1-dev (or expat-devel on some systems) is installed.

  • Insufficient Memory: Building the toolchain, especially gcc and gdb, can be memory-intensive. If you're on a system with limited RAM, consider adding swap space.
